# なつやすみロボずかん

夏休みの宿題を毎日クリアすると、ロボットが1体もらえるWebアプリです。

## 使い方（かんたん）

1. `index.html` をダブルクリックして開く
2. タイトル画面の「スタート」を押す（初回だけ、2人の名前・アイコン・お父さんのパスワードを設定）
3. 子どもがチェック → お父さんがパスワード入力 → **たっせいガチャ**が1回ぶん解放
4. ガチャ画面で引くと、カードがめくれてロボット獲得

## フォルダ構成

```
（リポジトリのルート）
├─ index.html      ← アプリ本体（これだけで動きます）
├─ robots.csv      ← 名前・説明の一覧（仕様書50体を反映済み）
├─ README.md
├─ images/         ← アプリが読む画像
│  ├─ 001.png 〜 050.png   ロボット50体
│  ├─ logo.png             タイトルロゴ（SYUKUDAI Building）
│  ├─ bg-title.jpg         背景：タイトル／こども選択
│  ├─ bg-home.jpg          背景：ホーム・宿題・ずかん・カレンダー・保護者
│  └─ bg-gacha.jpg         背景：ガチャ・獲得演出
├─ audio/          ← BGM（アプリが読む）
│  ├─ Beyond_The_Lab_Door.mp3
│  ├─ Build.mp3
│  └─ Nomal.mp3
├─ docs/           ← 企画・要件のメモ
└─ assets/         ← 素材の原本（アプリからは読み込まない）
   ├─ backgrounds/   加工前の背景・タイトルロゴ
   ├─ robots/        生成したロボット画像・検討用サンプル
   └─ spec/          ロボット50体_詳細仕様書.xlsx
```

`index.html` がルートにあるので、GitHub Pages はそのまま公開できます。
`assets/` は加工前の原本置き場です。ここを消してもアプリは動きます。

現在 `001.png` 〜 `005.png` は本番画像（カメル／ウサル／ハクビシン／カブト／エクレール）が入っています。
`006`〜`008`・`031`・`032`・`043`・`044` は仮のプレースホルダー、残りは未配置です。

## ロボット50体

名前・説明・レア度は `ロボット50体_詳細仕様書.xlsx` から取り込み済みです。**CSVを貼り付ける必要はありません。**

| レア度 | ID | 体数 | 内容 |
|---|---|---|---|
| ノーマル | 001–035 | 35 | 動物・昆虫・文房具・食べ物（仕様書の N） |
| レア | 036–042 | 7 | プリモ、ケーキル、ドラギア、フェニル、キリンガ、ユニオン、ペガード |
| ガチャレア | 043–050 | 8 | グリフォン、ケルベロ、キュウビ、オロチ、リヴァ、クラーケン、バハムート、シーサ |

仕様書は52体（N35＋R17）ありますが、アプリは50体構成のため **051 テング・052 オーガは予備**としています。
仕様書の N／R の区分はそのまま尊重し、R のうち上位の神話系8体をガチャレアに割り当てました。

名前や説明を変えたいときは、保護者メニューのCSV貼り付けで上書きできます。

## タイトル画面

起動すると必ずタイトル画面（**SYUKUDAI Building**）から始まります。

1. 森のラボの背景の上に、ロゴが 0.35秒後から約1.9秒かけてフェードイン（わずかに拡大しながら）
2. 2.1秒後に「スタート」ボタンが出て、点滅を始める
3. 押すと、初回は設定画面へ／2回目以降はこども選択へ

- ロゴは `images/logo.png`。元画像は市松模様が描き込まれていたため、透過PNGに切り出しています
- ロゴ画像が無いときは、文字組みのタイトルに自動で切り替わります
- タイトル画面はBGM（Beyond The Lab Door）の始まりでもあり、ブラウザの自動再生制限を解除する最初のタップの場にもなっています
- OSで「視差効果を減らす」を有効にしている場合は、アニメーションを省略して即表示します

## デザイン

明朝体（游明朝・ヒラギノ明朝）＋真っ黒の背景に、金と青のアクセント。
装飾は罫線と余白だけで組んでいます。

ロボット画像は白背景を抜いた透過PNGにしてあります（黒地に浮くように、足元の影も除去済み）。
本番の画像ができたら、同じファイル名で上書きしてください。

## 背景画像

真っ黒（#000）を下地に、**画面ごとに3種類の背景を出し分け**ています。

| 画面 | 使う画像 | 明るさ |
|---|---|---|
| タイトル・こども選択 | `images/bg-title.jpg` | 明るめ |
| ホーム・きょうの宿題・ずかん・カレンダー | `images/bg-home.jpg` | 明るめ |
| ガチャ・ロボット獲得演出 | `images/bg-gacha.jpg` | 明るめ |
| 保護者メニュー | `images/bg-home.jpg` | 暗め |

**子どもがさわる画面は明るく、保護者メニューだけ暗く**しています。
保護者メニューは長い文章や設定項目が多い作業画面なので、背景を沈めて文字を優先しました。

### 視認性のための3段構え

背景写真の上に白い明朝体を置くと文字が沈むため、次の3段階で下地を作っています。

1. **画像そのものを事前に減光** — 元画像はトーンマップ済み（コントラストを伸ばしてから暗く落とす）。単純に暗くすると質感まで消えるため、階調を伸ばしてから落としています。とくに「基本」は白いラボで元が明るいので強めに処理しています
2. **画面ごとの不透明度** — タイトル56%／基本50%／ガチャ68%／保護者30%
3. **2枚のスクリム** — 上端を濃く落とす縦グラデ（見出しの下地）＋中心から外へ落とす楕円グラデ（周辺の情報量を沈める）

さらにパネルは半透明の黒（子ども画面74〜76%／保護者66%＋ぼかし）にしてあるので、
**本文はパネルの上に乗る限り常に読めます**。背景を明るくしたぶん、パネルは少し濃くしてあります。
コントラスト比は本文で13:1以上、薄いグレーの補足文でも5.6:1以上を確保しています。

濃さを変えたいときは `index.html` の CSS 冒頭、`body[data-bg="…"]` の
`--bg-op`（画像の不透明度）／`--sc-in` `--sc-out`（楕円スクリム）／`--sc-top`（上端スクリム）を触ってください。

画像を差し替える場合は横長（1400〜1600px程度）を推奨します。画像が無い画面は、あるものを流用し、
1枚も無ければ真っ黒の背景のまま動きます。

## 画像について

- PNG（背景透過）／正方形／512px 程度を推奨
- **置いた画像だけが抽選対象**になります。まだ用意していない番号は出現しません
- 画像を足したら、保護者メニューの「🔄 画像を再スキャン」を押すか、ページを再読み込み
- フォルダを触れない環境では、保護者メニュー →「画像の追加登録」から直接アップロードもできます（その端末のみ有効）

## 音

### BGM

画面に合わせて3曲を切り替えます。切り替えは約0.5秒のフェードでつなぐので、ぶつ切りになりません。

| 画面 | 曲 |
|---|---|
| タイトル・こども選択 | `Beyond_The_Lab_Door.mp3` |
| ガチャ・ぜんぶ達成の完了画面 | `Build.mp3` |
| ホーム・きょうの宿題・ずかん・カレンダー・保護者メニュー | `Nomal.mp3` |

- 音量は42%。変えるときは `index.html` の `BGM_VOL` を触ってください
- 曲を差し替えるときは `audio/` に置いて `BGM_SRC` のファイル名を書き換えます
- 初期設定は **BGM ON**。保護者メニューの「BGM：ON／OFF」で切り替えられます
- ブラウザの制限で、**画面を一度タップするまで音は鳴りません**（仕様です）。タップした時点で、その画面の曲から始まります
- 同じ曲を使う画面のあいだを移動しても、曲は途切れず流れ続けます

完了画面（ぜんぶ達成）にお祝いとして `Build` を割り当てています。`Nomal` のほうが良ければ
`index.html` の `SCREEN_BGM` の `done:'gacha'` を `done:'normal'` に変えてください。

### タッチ音

ボタンやマスをさわると音が鳴ります。すべて内蔵の合成音なので、音声ファイルは不要です。

| 操作 | 音 |
|---|---|
| ふつうのボタン | 軽い「ポッ」 |
| 金色のメインボタン | 少し低めの2音 |
| もどる・とじる・キャンセル | 下がる音 |
| こども選択のカード | 開くような2音 |
| ずかんのマス・入力欄 | 軽い「ポッ」 |
| 宿題のチェック | ON「ピッ」／OFF「ポッ」（従来どおり） |

保護者メニューの「効果音：ON／OFF」でまとめて切れます。

## 名前・説明の一括入力

1. 保護者メニュー →「📋 ひな形をダウンロード」で `robots.csv` を取得
2. スプレッドシートで開いて `name` `desc` `rarity` を記入
3. スプレッドシートの内容を全選択してコピー
4. 保護者メニューの入力欄に貼り付けて「読み込む」

`rarity` は `normal` / `rare` / `gacha`（「ノーマル」「レア」「ガチャレア」でも可）。

## ルール

| 項目 | 内容 |
|---|---|
| 期間 | 2026/7/19 〜 8/20（33日間） |
| 達成 | 必須項目（日誌・サマースキル・ミライシード・チャレンジタッチ）が全部チェック |
| ★課題 | 作文・自主学習・あさがお観察などはボーナス扱い（達成条件には含まない） |
| ポイント | チャレンジタッチ達成で 2pt |
| 遅れ | 過ぎた未達成日はカレンダーで赤く残ります |

## 2つのガチャ

### たっせいガチャ

その日の宿題を全部おわらせて、保護者が承認すると **1回ぶん** 引けます。
ノーマル＋レアの42体から、まだ持っていないロボットが出ます。

- 承認するとそのままガチャ画面へ移動します
- すぐ引かなくても回数は消えません。あとでまとめて引けます
- 引ける状態のときは、下のナビの🎰に**赤い点**がつきます

### ボーナスガチャ

ポイントで引きます。ガチャレア8体から出ます。
**引くたびに必要なポイントが1ずつ増えます。**

| 回 | 必要pt | 累計 |
|---|---|---|
| 1回目 | 6 | 6 |
| 2回目 | 7 | 13 |
| 3回目 | 8 | 21 |
| 4回目 | 9 | 30 |
| 5回目 | 10 | 40 |
| 6回目 | 11 | 51 |
| 7回目 | 12 | 63 |
| 8回目 | 13 | 76 |

チャレンジタッチを毎日やると 2pt×33日＝66pt なので、**夏休み中におよそ7回**引ける計算です。
ガチャレア8体に対してちょうど引ききらないくらいのペースになります。

引ける回数は保存せず、承認済みの日と獲得済みロボットから毎回計算しています。
そのため端末間でデータを取り込んでも回数がズレません。

## 端末をまたいで使うとき

進捗データはブラウザごとに保存されるため、端末を変えるときは持ち出しが必要です。

**書き出し**：保護者メニュー →「📤 データをもちだす」→ `natsuyasumi_2026-08-05.json` が保存されます
**読み込み**：別端末で 保護者メニュー →「📥 データをよみこむ」→ そのファイルを選ぶ

読み込みは**マージ（良いとこ取り）**です。

- 達成した日・獲得したロボットは、両方の端末の記録が残る
- ポイントは重複計上されない（同じファイルを2回読んでも増えません）
- 名前やスケジュールの編集内容も引き継がれます

※ 保護者メニューからアップロードした画像は、ファイルサイズの都合でこのJSONには含まれません。`images/` フォルダ運用（または後述のURL公開）を推奨します。
※ JSONにはパスワードが平文で入ります。取り扱いにご注意ください。

## URLで公開する（GitHub Pages）

全端末で常に最新版が使え、画像の追加も一括反映されます。

1. GitHub Desktop でこのフォルダをプッシュ（Public）
2. GitHub のリポジトリ → Settings → Pages → Source を `Deploy from a branch`、`main` / `(root)` にして Save
3. 数分後 `https://ユーザー名.github.io/リポジトリ名/` で開けます
4. スマホでそのURLを開き、ブラウザの「ホーム画面に追加」でアプリのように使えます

`index.html` はリポジトリのルートにあるので、サブフォルダの指定は不要です。

ローカルのファイルをダブルクリックしても同じように動くので、どちらでも大丈夫です。

## 困ったとき

| 症状 | 対処 |
|---|---|
| ロボットがもらえない | ガチャ画面で引く方式になりました。ボタンが押せないときは `images/` に画像がない可能性大。保護者メニューの「画像を再スキャン」で枚数を確認 |
| パスワードを忘れた | 保護者メニューに入れないため、リセットが必要です（先にデータの持ち出しを） |
| 記録が消えた | ブラウザの「閲覧データの削除」で消えます。定期的に「データをもちだす」でバックアップを |
| 音が鳴らない | 画面を一度タップしてから。ブラウザの仕様で最初の操作まで音が出ません |
