# 夏休み宿題モンスターアプリ 仕様まとめ（回答反映版）

## 1. 基本

- **対象**：1年生・6年生の2人分を1アプリで（トップは子ども選択画面。名前とアイコンは初回起動時に入力）
- **データ**：きょうだいで完全に別々（図鑑・ポイント・進捗すべて独立）
- **期間**：7/19〜8/20（33日）／スケジュールはアプリに埋め込み＋保護者画面で編集可
- **端末**：レスポンシブ（タブレット・スマホ・PC いずれも）
- **表記**：漢字＋ふりがな（共通）
- **デザイン**：ポップでカラフル、大きなボタン
- **音**：効果音＋BGM（ON/OFF切り替え）
- **保存**：localStorage ＋ JSON書き出し／読み込み（バックアップ・端末移行用。詳細は §7）
- **配布**：**URL公開（GitHub Pages 等）**。ファイル構成は `index.html` ＋ `images/` のままなので、ローカルでダブルクリックしても動く二刀流にする

## 2. 達成判定と進捗

- 必須項目（にっし／日誌・サマースキル・タブレット等）が全部チェック → **その日の達成**
- ★印（作文・自主学習・あさがお観察・えにっき）は別枠のボーナス扱い
- 遅れた分は**赤く残り続ける**（自動繰り越しはしない。バッファ日で回収）
- 進捗表示は**カレンダーのスタンプ**（33日分。赤＝未達成）

## 3. モンスター獲得

### 通常
- 1日達成ごとに **1体**
- 抽選は**未所持から優先のランダム**（ダブりなし）
- **ただし、images に登録済みの画像からのみ抽選**（未登録のIDは出現候補から除外）

### ガチャ
- **チャレンジタッチ**の項目を達成すると **2ポイント**
- **6ポイント**でガチャ1回（＝3日分）
- ガチャからは**ガチャレア専用10体**が出る

### レア度
- 3段階：**ノーマル／レア／ガチャレア**
- 通常枠 40〜50体 ＋ ガチャレア10体

### 演出
- **カードがめくれる**アニメーション（レア度で光り方を変える）

## 4. パスワード（お父さんの承認）

- 流れ：子どもがチェック → **お父さんがパスワード入力** → モンスター獲得
- 形式：**英数字の自由文字列**（初回起動時に設定、設定画面で変更可）
- 間違い時：「ちがうよ」と表示して再入力（回数制限なし）

### 保護者メニュー
- 過去の日の承認・取り消し（さかのぼり修正）
- スケジュールの編集
- モンスターの手動プレゼント
- データのリセット
- ＋モンスター画像の追加登録

## 5. 画面一覧

1. **子ども選択**（起動直後）
2. **今日のチェック**
3. **図鑑**（グリッド。**未獲得は完全に非表示**）
4. **モンスター詳細**（大きい画像／なまえ／せつめい／レア度／獲得日）
5. **ガチャ**（ポイント残高＋「ひく」ボタン）
6. **カレンダー／進捗**
7. **ホーム（マイページ）**：相棒モンスターを表示
8. **保護者設定**（パスワードの奥）

## 6. モンスターデータ

| 項目 | 内容 |
|---|---|
| なまえ | こちらで用意（一覧に記入） |
| せつめい | こちらで用意 |
| レア度 | ノーマル／レア／ガチャレア |
| 獲得日 | 自動記録 |

### 画像
- **PNG（透過）／1体1枚**、正方形・512px程度を推奨
- `images/` に `001.png` `002.png` … と連番で配置
- **後から追加できる設計**（画像生成が開発に間に合わない前提）
  - 登録済みの画像のみが抽選対象になるため、**画像が1枚もなくてもアプリは動き**、追加した分だけ獲得候補が増える

## 7. 端末間のデータ移行（重要）

> 例：お母さんのスマホで進めていたが、その日はスマホが使えない → データを出して別端末で続きをやる

### 書き出し
- 保護者メニュー →「**データをもちだす**」
- `natsuyasumi_2026-08-05.json` のようなファイルとして保存
- LINE・AirDrop・メールなど、いつもの方法で別端末へ送る

### 読み込み
- 別端末で同じアプリを開く → 保護者メニュー →「**データをよみこむ**」→ ファイルを選ぶ
- 動作は「**マージ（良いとこ取り）**」
  - 達成した日・獲得したモンスターは、どちらの端末の記録も残る
  - ポイントは多い方を採用、使用済みガチャ回数は重複計上しない
  - 保護者パスワード・子どもの名前は、読み込み側に無ければ取り込む
- 読み込み後に「◯日ぶんの記録と◯たいのモンスターをとりこみました」と結果を表示

### 補足
- **URL公開にするので、アプリ本体は全端末で常に最新**（画像を追加したら全端末に自動反映）
- ただし **進捗データは端末ごとに独立**（localStorageのため）。だからこの書き出し／読み込みが必要
- 日常的には「主に使う1台」を決めて、サブ端末を使った日だけ移行するのが一番シンプル
